Inline Elements Test

This topic contains some inline elements like uicontrol , followed by cmdname, followed by parmname, followed by option, followed by codeph , followed by userinput , followed by apiname, followed by Wintitle, followed by an independent varname, and then a varname inside userinput.

Use of inline elements like, uicontrol introduce extra spaces. This becomes particularly problematic with commas: first , second , and third.

This happens with all inline elements, such as codeph :

this might be used for a list like sinusoid , sinusoidu, and cdt158. Also for cmdname, such as piconfig, piartool, and pidiag.

The Start, Stop, and Restart buttons on the toolbar are used to stop and start a service. This topic demonstrates how various inline elements appear, both within a sentence and as the last element in a sentence (before a punctuation mark).

Use the cmdname element for commands, such as piconfig.

Use the codeph element for bits of code or short messages, such as Do not jump.

Use the filepath element to show a directory or file name, such as C:\Program Files.
Use the italic element sparingly for emphasis, such as attention.
Use menucascade for file menus, such as File > New or File > Close.
Use the option element for values of parameters, such as True.
Use the parmname element for names of parameters, such as IsHappy.
Use the uicontrol element for words in the user interface, such as Field name.
Use the userinput element for text that users must type, such as piconfig -l.
Use the varname element for values that get substituted, such as Your Name. You might include varname elements inside other elements, such as userinput. For example, you might tell someone to enter First= YourFirstName, Last= YourLastName. Similarly, you might include varname elements inside codeph. For example, use the code For i=1 to CounterVariable .
Use the wintitle element to mark window titles, such as Browse Repository.
